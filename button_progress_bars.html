    Buttons & Progress Bars

    Use any of the available button classes to quickly create a styled button. Fancy larger or smaller buttons? Add .btn-lg, .btn-sm, or .btn-xs for additional sizes.


    Primary Buttons

    Primary buttons are used for the highest ranking actions on a page. They emphasize they key actions a user would take when navigating a certain page.



    <button type="button" class="btn btn-primary"></button>
<button type="button" class="btn btn-primary" disabled="disabled"></button>

<style>
.btn {
  display: inline-block;
  margin-bottom: 0;
  font-weight: normal;
  text-align: center;
  vertical-align: middle;
  touch-action: manipulation;
  cursor: pointer;
  white-space: nowrap;
  padding: 6px 12px;
  font-size: 14px;
  line-height: 1.5em;
  border-radius: 2px;
  -webkit-transition: all .25s ease;
  -moz-transition: all .25s ease;
  -ms-transition: all .25s ease;
  -o-transition: all .25s ease;
  transition: all .25s ease;
}

/* Active */
.btn-primary {
  background-color: #5bc0de;
  border: 1px solid #5bc0de;
  color: rgba(255,255,255,0.90);
}
.btn-primary:hover,
.btn-primary:focus,
.btn-primary:active {
  background-color: #50A9C3;
  border: 1px solid #50A9C3;
  color: #fff;
}
/* Inactive */
.btn-primary[disabled] {
  background-color: #8CD3E8;
  border: 1px solid #8CD3E8;
  color: rgba(255,255,255,0.90);
}
</style>

    Success Buttons

    Success buttons are used in multi-step forms (i.e. goal creation). They indicate a finish line. They should be the last button a user clicks in a multi-step process.



    <button type="button" class="btn btn-success"></button>
<button type="button" class="btn btn-success" disabled="disabled"></button>

<style>
/* Active */
.btn-success {
  border: 1px solid #82BB41;
  background-color: #82BB41;
  color: rgba(255,255,255,0.90);
}

.btn-success:hover,
.btn-success:focus,
.btn-success:active {
  border: 1px solid #70A438;
  background-color: #70A438;
  color: #fff;
}
/* Inactive */
.btn-success[disabled] {
  background-color: #A8CF79;
  border: 1px solid #A8CF79;
  color: rgba(255,255,255,0.70);
}
</style>

    Alternative Primary Buttons

    Alternative primary buttons are used when primary buttons cannot. This would occur when a button needs to be placed on an already highly saturated, colorful background. Both primary and secondary buttons are styled the same on saturated backgrounds.



    <button type="button" class="btn btn-primary"></button>
<button type="button" class="btn btn-primary" disabled="disabled"></button>

<style>
/* Active */
.btn-primary-alt {
  border: 1px solid #fff;
  background-color: rgba(255,255,255,0);
  color: rgba(255,255,255,0.90);
}
.btn-primary-alt:hover,
.btn-primary-alt:focus,
.btn-primary-alt:active {
  background-color: #fff;
  color: #5bc0de; /* Parent background color */
}
/* Inactive */
.btn-primary-alt[disabled] {
  background-color: rgba(255,255,255,0);
  border: 1px solid rgba(255,255,255,0.70);
  color: rgba(255,255,255,0.70);
}
</style>

    Secondary Buttons

    Secondary buttons are used for secondary actions. When paired with a primary button, the secondary button will always be on the left.


    • Action
    • Another action
    • Something else here
    • Separated link


    <!-- Dropdown Secondary Button -->
<div type="button" class="btn-group">
  <button type="button" class="btn btn-secondary dropdown-toggle" data-toggle="dropdown" aria-expanded="false">
    Dropdown <span class="caret"></span>
  </button>
  <ul class="dropdown-menu" role="menu">
    <li><a href="#">Action</a></li>
    <li><a href="#">Another Action</a></li>
    <li><class="divider"></li>
    <li><a href="#">Separated Link</a></li>
  </ul>
</div>

<!-- Secondary Button w/ icon -->
<button type="button" class="btn btn-secondary">
  <span class="glyphicon glyphicon-pencil"></span> Edit
</button>

    Progress Bars

    Progress bars is used for displaying the progress of goals. When a goal is on track, the progress bar is green. When a goal is 25% behind schedule, the progress bar is yellow. When a goal is 50% behind schedule, the progress bar is red.

    10% Complete (completed)

    40% Complete (expected)
    10% Complete (completed)
    90% Complete (expected)
